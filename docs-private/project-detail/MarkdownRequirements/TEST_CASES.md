# 🤖 テスト実装指示書 (Implementation Agent Prompt)

> **実装AIエージェントへの指示 (Prompt for Implementation Agent):**
> あなたの任務は、このドキュメントに記載されたテストケースを **Vitest**, **React Testing Library**, **MSW**, および **Playwright** を用いて実際の実装コードに落とし込むこと（How）です。
> **Testing Trophy（テストのトロフィー）**戦略に基づき、単なる Unit テストではなく、状態管理（React Query）と連動した **Integration（結合テスト）** を最優先で実装してください。

## 🛠️ 技術スタックと検証レイヤーの定義

- **[Unit]**: 純粋なロジックや個別部品の検証。モックを活用し、特定の条件分岐を高速に回す。
- **[Integration] (最重要)**: コンポーネントと本物の状態管理（React Query / Zustand）を連動させる。安易な `vi.mock` によるフックの無効化を避け、実際のキャッシュ更新フローを物理的に証明する。
- **[E2E]**: Playwright。本物のブラウザとDBを使用し、ログインから投稿までの「太いユーザーフロー」を完走させる。
- **[Backend]**: Supabase RLS / DB制約。システムとしての最終的な防波堤。
