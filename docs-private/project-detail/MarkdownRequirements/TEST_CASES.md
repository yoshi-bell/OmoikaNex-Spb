## 🛠️ 技術スタックと実装ルール

- **Unit / Integration Test:** `Vitest` + `@testing-library/react`
- **API Mocking:** `MSW (Mock Service Worker)` を使用し、Supabaseクライアント（`@supabase/ssr`）へのリクエストをインターセプトする。
- **E2E Test:** `Playwright`
- **ディレクトリ構成:** `src/features/[feature]/components/__tests__/` 等、対象ファイルの近傍に配置する。
- **絶対厳守:** テストコード内で直接DBへ書き込まないこと。必ず MSW で API レスポンスをモックすること。

> **【重要: エラーハンドリングと MSW モックのルール】**
> Supabaseから返却されるエラー（AuthErrorやPostgRESTError等）は、必ずフロントエンドの **Repository層でキャッチ・変換** し、UIコンポーネントには Laravel 互換の馴染み深いエラーオブジェクトとして伝える設計です。MSW ではこの「変換前のエラー」を意図的に発生させ、UIの挙動をテストしてください。
